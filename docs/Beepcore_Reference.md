# Beepcore MML リファレンス

Beepcore は、エンジンのメインループに同期して動作する、WebAudio API ベースの音響合成システムです。
MML（Music Macro Language）風の文字列を使用して、手軽にメロディや効果音を作成できます。

---

## 1. MML コマンド一覧

Beepcore の MML 解析エンジンは以下のコマンドに対応しています。大文字・小文字は区別されません。

| コマンド | 内容 | 詳細 |
| :--- | :--- | :--- |
| **A 〜 G** | 音名 | 音階（ドレミファソラシ）を再生します。 |
| **#** / **+** | シャープ | 音名の直後に付けて半音上げます（例：`C#`, `F+`）。 |
| **R** | 休符 | 指定された長さの間、音を止めます。 |
| **O** | オクターブ | `O0` 〜 `O8` で指定します。デフォルトは `O4` です。 |
| **L** | 音長 | デフォルトの音の長さを指定します（例：`L4` は四分音符）。 |
| **V** | 音量 | `V0` 〜 `V15` で指定します。音量は `n/15.0` で計算されます。 |
| **.** | 付点 | 音符や休符の直後に付け、長さを 1.5 倍にします（例：`C4.`）。 |
| **T** | テンポ | MML内での記述は可能ですが、現在は解析時に無視されます。 |

### 音の長さについて
音符の直後に数字を書くことで、その音だけの長さを指定できます。
数字は「分音符」の分母を表します（例：`C8` は八分音符、`C1` は全音符）。

### テンポとミリ秒の対応（目安）
`playScore` の `interval` 引数は「四分音符（L4）1つ分の長さ」をミリ秒で指定します。
一般的な音楽のテンポ（BPM: Beats Per Minute）からミリ秒へ変換する計算式は以下の通りです。

**計算式: `interval (ms) = 60000 / BPM`**

| BPM (テンポ) | 四分音符の長さ (ms) | 特徴・目安 |
| :--- | :--- | :--- |
| 60 | 1000 | 1秒に1拍。非常にゆったり。 |
| 100 | 600 | 標準的な歩行テンポ。 |
| **120** | **500** | **標準的。2拍で1秒。** |
| 150 | 400 | 行進曲のような軽快なテンポ。 |
| 180 | 333 | アップテンポな楽曲。 |
| 200 | 300 | 非常に速い。激しいアクション等。 |

## 2. システム設定と拡張機能

### 波形タイプの設定
`setOscType(type)` メソッドで基本波形を変更できます。
- `'square'` (デフォルト): ファミコン風の矩形波
- `'sine'`: 柔らかなサイン波
- `'sawtooth'`: 鋭いノコギリ波
- `'triangle'`: 落ち着いた三角波

### LFO (ビブラート) の設定
`setLFO(freq, type, depth)` で音程を揺らすことができます。
- `freq`: 揺れの速さ (Hz)
- `depth`: 揺れの深さ (周波数の変動幅)

## 3. 実装サンプル

### 基本的な再生
```javascript
// 四分音符の基準時間を 200ms として「ドレミ」を演奏
engine.sound.playScore("C D E", 200, performance.now());
```

### MML を駆使したメロディ
```javascript
// オクターブ変更、音長、付点を含む例
const melody = "O5 L8 C D E. F16 G4 R4 O4 G A B O5 C";
engine.sound.playScore(melody, 150, performance.now());
```

### ビブラートを効かせたシンセ音
```javascript
engine.sound.beep.setOscType('triangle');
engine.sound.beep.setLFO(6, 'sine', 10); // 6Hz で揺らす
engine.sound.playScore("O6 V10 C2. R4", 100, performance.now());
```

## 4. 内部仕様と注意点

1. **メインループ同期**: `AudioManager.step(now)` が毎フレーム呼ばれることで、経過時間に応じた正確な音程の切り替えが行われます。
2. **チャタリング防止**: 100ms 以内に全く同じ文字列が再生リクエストされた場合、二重発音を防ぐために無視されます。
3. **ノイズ対策**: 直値代入による高速な発音を行いつつ、音の終了時には 5ms の極短フェードアウト（Ramp）を行い、プツッというノイズを軽減しています。
4. **音源プール**: 同時発音数に対応するため、内部でオシレーターのインスタンスをプール管理しています。
